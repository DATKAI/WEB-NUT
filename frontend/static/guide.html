Инструкция

Как пользоваться панелью

Пошаговое руководство для начала работы

1
Вход в систему
Откройте браузер и перейдите по адресу панели. Введите логин и пароль.
http://192.168.10.3:8000
ℹ️ Логин по умолчанию: admin / пароль задаётся при установке и хранится в файле /opt/nut-monitor/.env
2
Дашборд — главная страница
После входа вы попадаете на дашборд. Здесь отображаются:
  • Карточки всех ИБП с зарядом, нагрузкой и напряжением
  • Кнопки быстрых команд: тест батареи, включение/выключение звука
  • Таблица подключённых клиентов с их статусом
  • Последние события
⚠️ Данные обновляются автоматически каждые 3 секунды — обновлять страницу не нужно
3
Подключение Windows клиентов
Перейдите в раздел Клиенты → Windows. Для подключения Windows серверов есть два варианта:
  • WinNUT-Client — GUI программа с иконкой в трее
  • NUT Monitor Service — служба Windows (работает без входа пользователя)
Для службы (рекомендуется):
1. Скачайте nut-service-install.zip из панели 2. Распакуйте все файлы в одну папку 3. Запустите nut-service-install.bat от администратора 4. Скрипт скачает WinSW и установит службу автоматически
ℹ️ Служба запускается от имени SYSTEM — работает даже если никто не вошёл в Windows. Лог: C:\NUT-Monitor\nut-monitor.log
4
Подключение Linux клиентов
Перейдите в раздел Клиенты → Linux. Скопируйте и выполните команды по порядку:
1. apt install -y nut-client 2. Создать /etc/nut/nut.conf (MODE=netclient) 3. Создать /etc/nut/upsmon.conf (скопировать из панели — данные уже подставлены) 4. systemctl enable --now nut-client 5. Установить heartbeat скрипт (шаг 6 в панели)
⚠️ Heartbeat скрипт нужен чтобы клиент отображался в таблице мониторинга. Без него upsmon работает, но не виден в панели.
5
Настройка уведомлений
Перейдите в раздел Уведомления. Доступны два канала:
  • Telegram — укажите Bot Token и Chat ID
  • Email (SMTP) — укажите данные почтового сервера
Выберите какие события уведомлять:
  • Переход на батарею (OB)
  • Низкий заряд (LB)
  • Восстановление питания (OL)
ℹ️ После настройки нажмите «Тест» — придёт пробное сообщение
6
Настройка NUT Сервера
Раздел NUT Сервер содержит три вкладки:
  • Статус — состояние служб nut-server и nut-monitor, кнопки перезапуска
  • Конфигурация — NUT admin логин/пароль (для команд), порог низкого заряда, интервал опроса
  • NUT Пользователи — управление пользователями upsd (slave для клиентов, master для управления)
⚠️ Для выполнения команд (тест батареи, звук) необходимо указать NUT admin логин/пароль в Конфигурации — пользователь с ролью master и instcmds=all
7
Обновление панели
Для обновления панели до последней версии выполните на сервере:
cd /root/WEB-NUT && bash update.sh
ℹ️ Скрипт делает git pull, обновляет зависимости и перезапускает службу. База данных и настройки сохраняются.

Расшифровка статусов ИБП

Что означают значки на дашборде

Статус Код NUT Описание Действие
✅ Сеть OL Питание от сети, всё в норме Ничего не нужно
⚠️ Батарея OB Питание от батареи — пропало сетевое питание Проверить электропитание
🔴 Низкий заряд LB Батарея почти разряжена — скоро выключение Клиенты начнут выключение!
⚡ Заряжается CHRG Батарея заряжается после работы на батарее Ничего не нужно
📉 Срез напр. TRIM ИБП снижает высокое входное напряжение Следить за напряжением
📈 Подъём напр. BOOST ИБП повышает низкое входное напряжение Проверить качество питания
🔴 Shutdown! FSD Принудительное выключение — команда от сервера Все клиенты выключаются
🔋 Замени АКБ RB ИБП сигнализирует о необходимости замены батареи Заменить аккумулятор

Часто задаваемые вопросы

Почему после входа нужно обновлять страницу? ▼
Это было исправлено. Теперь после успешного входа происходит автоматический редирект на дашборд. Если проблема осталась — очистите кэш браузера (Ctrl+Shift+R).
Клиент не появляется в таблице мониторинга ▼
Для Windows: убедитесь что служба NUT-Monitor запущена (services.msc). Проверьте лог C:\NUT-Monitor\nut-monitor.log.

Для Linux: убедитесь что установлен heartbeat скрипт (шаг 6 в инструкции) и таймер активен: systemctl status nut-heartbeat.timer

Клиент пропадает из таблицы если не присылал данные более 5 минут.
Ошибка ERR ACCESS-DENIED при выполнении команд ▼
Необходимо создать NUT пользователя с правами на команды и указать его в настройках панели.

1. NUT Сервер → NUT Пользователи → Добавить: логин upsadmin, роль master, instcmds: all
2. NUT Сервер → Конфигурация → NUT admin логин/пароль → Сохранить
NUT сервер не запускается после «Применить конфиг» ▼
Выполните на сервере:
echo "MODE=netserver" > /etc/nut/nut.conf
systemctl start nut-server

Начиная с текущей версии панель автоматически проверяет и восстанавливает MODE=netserver перед каждым перезапуском.
Как обновить панель? ▼
Выполните на сервере Proxmox:
cd /root/WEB-NUT && bash update.sh

Скрипт загружает последние изменения с GitHub, обновляет зависимости и перезапускает службу. Все настройки и база данных сохраняются.
Как удалить NUT Monitor Service с Windows клиента? ▼
Запустите PowerShell от администратора и выполните:
powershell -File nut-service-install.ps1 -Uninstall

Или вручную:
C:\NUT-Monitor\NUT-Monitor.exe stop
C:\NUT-Monitor\NUT-Monitor.exe uninstall
Что означает «OL CHRG» — ИБП заряжается, но заряд уже 100%? ▼
Это нормально для некоторых моделей ИБП. Некоторые устройства продолжают показывать статус CHRG даже когда батарея полностью заряжена — пока через них течёт ток поддержания заряда. Со временем статус CHRG должен исчезнуть и останется только OL.
